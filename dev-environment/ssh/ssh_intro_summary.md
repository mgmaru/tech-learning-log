# SSH入門まとめ

## 1. SSHとは

SSHとは、**離れた場所にあるコンピューターへ安全にログインして、コマンドで操作するための仕組み**です。

正式名称は **Secure Shell** です。

イメージとしては、次のようなものです。

```text
自分のPC
  ↓ SSH接続
離れた場所にあるPC / サーバー
  ↓
コマンドで操作する
```

たとえば、自宅のノートPCからクラウド上のLinuxサーバーへ接続したり、外出先のPCから家のPCへ接続したりできます。

SSHは、Web開発やサーバー運用では非常によく使われる基本技術です。

---

## 2. SSHでできること

SSHを使うと、接続先のPCやサーバーに入って、ターミナル上でコマンドを実行できます。

代表的には、次のような操作ができます。

```bash
ls
cd
git pull
docker compose up -d
systemctl restart nginx
tail -f app.log
```

つまり、SSHは「画面を直接操作する」というより、**コマンドを使って遠隔操作する仕組み**です。

---

## 3. SSHの基本構成

SSHでは、主に次の2つの役割があります。

| 役割 | 意味 |
|---|---|
| SSHクライアント | 接続する側 |
| SSHサーバー | 接続される側 |

たとえば、外出先のノートPCから家のPCに接続する場合は、次のようになります。

```text
外出先のノートPC
  = SSHクライアント

家のPC
  = SSHサーバー
```

SSHクライアントからSSHサーバーへ接続することで、接続先のPCをコマンド操作できます。

---

## 4. SSHを使う主な場面

SSHは、以下のような場面で使われます。

| 使用場面 | 具体例 |
|---|---|
| サーバーにログインする | VPS、クラウドサーバー、社内サーバーに入る |
| アプリをデプロイする | サーバー上で `git pull` してアプリを更新する |
| Dockerを操作する | リモートサーバーでコンテナを起動・停止する |
| ログを確認する | 本番環境のログを確認する |
| ファイル転送をする | `scp` や `sftp` でファイルを送る |
| GitHubと通信する | SSHキーを使って `git push` / `git pull` する |
| Raspberry Piを操作する | 画面なしで遠隔操作する |

特に、個人開発でVPSやクラウドサーバーを使うようになると、SSHはかなり重要になります。

---

## 5. SSHのメリット

### 5.1 安全に通信できる

SSHの大きな特徴は、通信内容が暗号化されることです。

```text
自分のPC ←→ 接続先サーバー
        通信内容が暗号化される
```

そのため、通信途中で内容を盗み見されても、パスワードやコマンド内容を簡単には読み取られません。

---

### 5.2 遠隔から操作できる

SSHを使えば、物理的にそのPCの前にいなくても操作できます。

たとえば、自宅にあるPCやクラウド上のサーバーを、外出先から操作できます。

```text
外出先のノートPC
  ↓ SSH
自宅PC / VPS / クラウドサーバー
```

---

### 5.3 軽い

SSHは基本的に文字情報だけをやり取りします。

そのため、リモートデスクトップのように画面映像を送る方式よりも軽いです。

```text
SSH:
  文字情報中心
  → 軽い

リモートデスクトップ:
  画面映像を転送
  → SSHより重い
```

回線が弱い環境でも、SSHは比較的安定して使いやすいです。

---

### 5.4 自動化しやすい

SSHでは、接続先にコマンドだけを実行させることもできます。

例:

```bash
ssh user@example.com "docker compose restart"
```

このように、リモートサーバーにログインしてから手動で操作するだけでなく、コマンド実行を自動化しやすいのもメリットです。

---

## 6. SSHの認証方法

SSHで接続するには、接続先が「この人はログインしてよい」と判断する必要があります。

代表的な認証方法は以下の2つです。

| 認証方法 | 内容 |
|---|---|
| パスワード認証 | ユーザー名とパスワードでログインする |
| 公開鍵認証 | 秘密鍵と公開鍵のペアを使ってログインする |

---

## 7. 公開鍵認証とは

SSHでは、**公開鍵認証**がよく使われます。

公開鍵認証では、次の2つの鍵を使います。

| 鍵 | 置く場所 | 扱い |
|---|---|---|
| 秘密鍵 | 自分のPC | 絶対に他人に渡さない |
| 公開鍵 | 接続先サーバー | 登録しておく |

イメージは次の通りです。

```text
自分のPC
  └── 秘密鍵

接続先サーバー
  └── 公開鍵
```

接続時に、秘密鍵と公開鍵が対応していることを確認できると、ログインが許可されます。

### 重要

秘密鍵は、自分だけが持つ重要なファイルです。

秘密鍵が他人に漏れると、その鍵でログインされる危険があります。

そのため、秘密鍵は絶対に他人に渡してはいけません。

---

## 8. SSHのOS制限

SSH自体は、特定のOS専用の仕組みではありません。

SSHは通信プロトコルなので、主要なOSで利用できます。

| OS | SSHクライアント | SSHサーバー |
|---|---|---|
| Linux | 使える | 使える |
| macOS | 使える | 使える |
| Windows 10 / 11 | 使える | 使える |
| Raspberry Pi OS | 使える | 使える |
| Android / iOS | アプリ経由で使える | 特殊用途向け |

そのため、次のような組み合わせも可能です。

```text
Windows → Linux
Mac → Linux
Windows → Mac
Mac → Windows
Linux → Windows
iPad → Linuxサーバー
```

重要なのは、OSの種類よりも次の3つです。

```text
1. 接続する側にSSHクライアントがあること
2. 接続される側でSSHサーバーが動いていること
3. ネットワーク的に到達できること
```

---

## 9. 家のPCに外出先からSSH接続できるのか

結論としては、可能です。

家のPCにSSHサーバーを設定しておけば、外出先のノートPCから家のPCへSSH接続できます。

```text
外出先のノートPC
  ↓
インターネット
  ↓
自宅ルーター
  ↓
家のPC
```

ただし、外出先から家のPCへ接続する場合は、家のネットワークに外から到達するための仕組みが必要です。

---

## 10. 同じWi-Fi内でSSHする場合

家の中で、同じWi-Fiに接続しているPC同士であれば、比較的簡単です。

```text
ノートPC
  ↓
自宅Wi-Fi / ルーター
  ↓
家のPC
```

例:

```bash
ssh user@192.168.x.x
```

この場合、`192.168.x.x` は家のPCのローカルIPアドレスです。

---

## 11. 外出先からSSHする場合

外出先から家のPCに接続する場合は、少し難しくなります。

なぜなら、通常の家庭内PCはインターネットから直接見えない場所にあるからです。

外出先から接続するには、主に次のような方法があります。

| 方法 | 内容 | 注意点 |
|---|---|---|
| ポート開放 | ルーターでSSH通信を家のPCに通す | セキュリティ注意 |
| VPN | 自宅ネットワークに安全に入る | 設定が必要 |
| Tailscale / ZeroTier | VPNのような接続を簡単に作る | 初心者にも比較的扱いやすい |

初心者が安全に始めるなら、いきなりポート開放するよりも、Tailscaleのような仕組みを使う方が安全寄りです。

---

## 12. SSHとリモートデスクトップの違い

SSHと似たような話として、リモートデスクトップがあります。

両者の違いは、操作方法です。

| 比較 | SSH | リモートデスクトップ |
|---|---|---|
| 操作方法 | コマンドで操作 | 画面を見ながら操作 |
| 見えるもの | ターミナル | 相手PCのデスクトップ画面 |
| 向いている用途 | サーバー管理、開発、ログ確認 | GUIアプリ操作、普段のPC操作 |
| 通信量 | 少ない | 多い |
| 軽さ | 軽い | SSHより重い |
| 初心者のわかりやすさ | 最初は難しめ | 直感的でわかりやすい |

---

## 13. SSHが向いている場面

SSHは、次のような場面に向いています。

```text
Linuxサーバーにログインする
Dockerを再起動する
GitHubからコードをpullする
ログを確認する
設定ファイルを編集する
コマンド操作で済む作業をする
```

特にサーバー管理や開発作業では、SSHが非常に便利です。

---

## 14. リモートデスクトップが向いている場面

一方で、リモートデスクトップは次のような場面に向いています。

```text
家のPCの画面をそのまま見たい
マウスで操作したい
WindowsやMacのGUIアプリを使いたい
Clip Studio、Premiere、ブラウザなどを操作したい
ファイルを目で確認しながら整理したい
```

外出先から家のPCを「普段通りに操作したい」なら、SSHよりリモートデスクトップの方が直感的です。

---

## 15. 使い分けの考え方

SSHとリモートデスクトップは、どちらが絶対に良いというものではありません。

目的によって使い分けます。

```text
コマンドで操作したい
  → SSH

画面を見ながら操作したい
  → リモートデスクトップ

安全に外出先から使いたい
  → Tailscale / VPN などを併用
```

個人利用で「家のPCを外から使いたい」場合は、次のような使い分けが現実的です。

| 目的 | おすすめ |
|---|---|
| サーバー・開発環境を操作する | SSH |
| 家のWindows PCを普通に操作する | リモートデスクトップ |
| 安全に外出先から接続する | Tailscale + SSH / リモートデスクトップ |

---

## 16. Tera Termとは

Tera Termは、Windowsでよく使われるSSHクライアントソフトです。

WindowsからLinuxサーバーやネットワーク機器へSSH接続するときに使えます。

Tera Termを使うと、画面上で次の情報を入力して接続できます。

```text
ホスト名
ユーザー名
パスワード
秘密鍵
ポート番号
```

コマンドで `ssh user@example.com` と入力する代わりに、GUIで接続情報を指定できるイメージです。

---

## 17. Tera Termのメリット

Tera Termには次のようなメリットがあります。

| メリット | 内容 |
|---|---|
| GUIで接続しやすい | ホスト名や秘密鍵を画面で指定できる |
| 日本語情報が多い | 日本の現場でもよく使われている |
| SSH鍵認証に対応 | 秘密鍵を使ったログインができる |
| ログ保存がしやすい | 作業ログをファイルに残しやすい |
| マクロ機能がある | 定型作業を自動化できる |
| シリアル接続にも使える | ネットワーク機器やマイコンにも使える |

SSHに慣れ始めた段階では、Tera Termを使うと接続の流れを理解しやすいです。

---

## 18. Tera TermとWindows標準sshの違い

Windowsでは、現在はPowerShellやWindows Terminalから `ssh` コマンドを使うこともできます。

| 比較 | Tera Term | Windows Terminal / PowerShell |
|---|---|---|
| 操作感 | GUI寄り | コマンド寄り |
| 接続しやすさ | 初心者にわかりやすい | 慣れると速い |
| 鍵ファイル指定 | 画面で選べる | `-i` オプションなどを使う |
| ログ保存 | やりやすい | 工夫が必要 |
| Gitや開発作業との相性 | 主用途ではない | 相性が良い |
| VS Code連携 | 主用途ではない | Remote SSHと相性が良い |

学習初期や手動接続ではTera Termが便利です。

一方で、開発作業では、最終的にWindows Terminal、PowerShell、VS Code Remote SSHにも慣れておくと便利です。

---

## 19. SSH接続の具体例

### 19.1 基本の接続

```bash
ssh user@example.com
```

意味は次の通りです。

| 部分 | 意味 |
|---|---|
| `ssh` | SSH接続するコマンド |
| `user` | 接続先のユーザー名 |
| `example.com` | 接続先のホスト名またはIPアドレス |

---

### 19.2 IPアドレスで接続する例

```bash
ssh user@192.168.1.10
```

これは、ローカルネットワーク内の `192.168.1.10` というPCへ、`user` というユーザーで接続する例です。

---

### 19.3 秘密鍵を指定する例

```bash
ssh -i ~/.ssh/id_ed25519 user@example.com
```

`-i` は、使用する秘密鍵ファイルを指定するオプションです。

---

### 19.4 ポート番号を指定する例

```bash
ssh -p 2222 user@example.com
```

通常、SSHは22番ポートを使います。

ただし、サーバー側の設定によっては、別のポート番号を使う場合があります。

---

## 20. SSHでファイル転送する方法

SSHの仕組みを使って、ファイル転送もできます。

代表的なのは `scp` や `sftp` です。

### 20.1 scpの例

```bash
scp ./app.zip user@example.com:/home/user/
```

これは、手元の `app.zip` を接続先サーバーの `/home/user/` に送る例です。

---

### 20.2 sftpの例

```bash
sftp user@example.com
```

SFTPを使うと、SSH経由で安全にファイルを送受信できます。

---

## 21. SSHを安全に使うポイント

SSHは便利ですが、設定を誤ると危険もあります。

特に、外部から接続できるようにする場合は注意が必要です。

### 21.1 鍵認証を使う

可能であれば、パスワード認証よりも公開鍵認証を使う方が安全です。

```text
パスワード認証
  → パスワードが漏れると危険

公開鍵認証
  → 秘密鍵を持っている人だけがログインできる
```

---

### 21.2 秘密鍵を漏らさない

秘密鍵は絶対に他人に渡してはいけません。

また、GitHubなどのリポジトリに誤ってコミットしないように注意が必要です。

---

### 21.3 不要にポート開放しない

自宅PCに外出先からSSHしたい場合、ルーターのポート開放を使う方法もあります。

しかし、SSHポートをインターネットに公開すると、攻撃対象になる可能性があります。

そのため、初心者は以下のような構成の方が安全寄りです。

```text
外出先PC
  ↓
Tailscale / VPN
  ↓
自宅PC
```

---

### 21.4 パスワード認証を無効化する

公開鍵認証が使えるようになったら、サーバー側でパスワード認証を無効化することも検討します。

ただし、設定ミスをすると自分もログインできなくなる可能性があるため、慎重に行う必要があります。

---

## 22. SSHを学ぶおすすめの順番

SSHを初めて学ぶなら、次の順番がわかりやすいです。

```text
1. 同じWi-Fi内のPCにSSH接続してみる
2. SSHクライアントとSSHサーバーの関係を理解する
3. パスワード認証で接続の流れを理解する
4. 公開鍵認証を試す
5. scp / sftp でファイル転送を試す
6. VPSやクラウドサーバーにSSH接続してみる
7. Tailscaleなどを使って安全なリモート接続を試す
```

いきなり外部公開やポート開放から始めるより、まずは同じWi-Fi内で練習するのがおすすめです。

## 23. SSHの全体像まとめ

SSHは、次のように理解するとわかりやすいです。

```text
SSH = 離れたPCやサーバーに安全に入って、
      コマンドで操作するための仕組み
```

ポイントは次の通りです。

| ポイント | 内容 |
|---|---|
| SSHの目的 | 遠隔のPCやサーバーを安全に操作する |
| 操作方法 | 基本はコマンド操作 |
| 接続する側 | SSHクライアント |
| 接続される側 | SSHサーバー |
| 強み | 軽い、安全、自動化しやすい |
| よく使う場面 | サーバー管理、開発、ログ確認、デプロイ |
| GUI操作したい場合 | リモートデスクトップの方が向いている |
| Windowsで使うなら | Tera TermやWindows Terminalが候補 |

## 24. 最終的な使い分け

最後に、SSH・リモートデスクトップ・Tera Termの関係を整理します。

| 道具 | 役割 | 向いている用途 |
|---|---|---|
| SSH | コマンドで遠隔操作する仕組み | サーバー管理、開発、ログ確認 |
| リモートデスクトップ | 画面を見ながら遠隔操作する仕組み | GUIアプリ操作、普段のPC操作 |
| Tera Term | Windows向けSSHクライアント | SSH接続、作業ログ保存、手動作業 |
| Windows Terminal | Windows標準のターミナル | 開発作業、Git、CLI操作 |
| VS Code Remote SSH | VS CodeからSSH先を開発環境として使う | リモート開発 |

SSHは、最初は少し難しく見えますが、サーバーや開発環境を扱う上では非常に重要な基礎技術です。

リモートデスクトップと比較しながら理解すると、SSHの役割がかなり見えやすくなります。
