# Tailscale入門まとめ

## 目的

このドキュメントは、Tailscaleをまだ使ったことがない人向けに、基本情報を整理した学習メモです。

特に、これまで議論してきた以下の内容との関係がわかるようにまとめます。

```text
SSH
VPN
VPS
自宅PCへのリモート接続
リモートデスクトップ
ポート開放
```

Tailscaleは、SSHやリモートデスクトップを安全に使いやすくするための選択肢として、とても便利です。

---

# 1. Tailscaleとは

Tailscaleは、複数のPC・スマホ・サーバーを、同じプライベートネットワーク内にいるようにつなげるサービスです。

一言でいうと、次のように理解できます。

```text
Tailscale = 自分の端末だけが参加できる、かんたんなプライベートネットワークを作るサービス
```

たとえば、次のような端末を持っているとします。

```text
自宅のデスクトップPC
外出用ノートPC
スマホ
VPS
```

これらすべてにTailscaleをインストールして、同じアカウントでログインすると、端末同士が安全につながります。

```text
          Tailscaleの仮想ネットワーク
        ┌────────────────────┐
        │ 自宅PC              │
        │ 外出用ノートPC      │
        │ スマホ              │
        │ VPS                 │
        └────────────────────┘
```

---

# 2. TailscaleはVPNなのか

Tailscaleは、VPNに近いサービスです。

より正確には、WireGuardというVPN技術をベースにした、メッシュVPNのような仕組みです。

ただし、通常のVPNよりも設定が簡単です。

```text
一般的なVPN:
  VPNサーバーを用意する
  ルーターの設定をする
  証明書や鍵を管理する
  ルーティングを設定する

Tailscale:
  各端末にインストールする
  同じアカウントでログインする
  Tailscale IPで接続する
```

そのため、Tailscaleは「VPNの難しい部分をかなり簡単にしたもの」と考えるとわかりやすいです。

---

# 3. Tailscaleで何ができるのか

## 3.1 外出先から自宅PCに接続できる

Tailscaleを使うと、外出先のノートPCから自宅PCへ接続しやすくなります。

```text
外出先ノートPC
  ↓ Tailscale
自宅PC
```

この状態で、SSHやリモートデスクトップを使えます。

```text
外出先ノートPC
  ↓ Tailscale
自宅PC
  ↓ SSH
コマンド操作
```

または、

```text
外出先ノートPC
  ↓ Tailscale
自宅PC
  ↓ リモートデスクトップ
画面操作
```

---

## 3.2 VPSと自宅PCを安全につなげられる

VPSにもTailscaleを入れると、自宅PC・ノートPC・VPSを同じ仮想ネットワークに参加させられます。

```text
自宅PC
  ├── Tailscale
外出先ノートPC
  ├── Tailscale
VPS
  └── Tailscale
```

この場合、VPSの公開IPではなく、Tailscale上のIPでアクセスできます。

```bash
ssh user@VPSのTailscale IP
```

---

## 3.3 ポート開放なしで接続しやすい

通常、外出先から自宅PCにSSHやリモートデスクトップで接続しようとすると、自宅ルーターでポート開放が必要になることがあります。

```text
外出先PC
  ↓
インターネット
  ↓
自宅ルーターでポート開放
  ↓
自宅PC
```

しかし、ポート開放は設定ミスをすると危険です。

Tailscaleを使うと、次のように接続できます。

```text
外出先PC
  ↓ Tailscale
自宅PC
```

この構成では、自宅PCのSSHポートやリモートデスクトップ用ポートを直接インターネットに公開しなくて済みます。

---

# 4. 通常のVPNとの違い

## 4.1 従来のVPN

従来のVPNは、VPNサーバーに接続して、そのネットワークへ入る構成が一般的です。

```text
外出先PC
  ↓
VPNサーバー
  ↓
自宅ネットワーク / 会社ネットワーク
```

この場合、VPNサーバーが中心にあります。

---

## 4.2 Tailscaleのイメージ

Tailscaleは、端末同士を直接つなぐメッシュ型のイメージです。

```text
外出先PC  ───── 自宅PC
    │             │
    └──── VPS ────┘
```

もちろん、ネットワーク状況によって中継が入ることもありますが、利用者目線では「端末同士が同じプライベートネットワークにいる」ように扱えます。

---

## 4.3 比較表

| 項目 | 一般的なVPN | Tailscale |
|---|---|---|
| 基本構成 | VPNサーバー中心 | 端末同士をつなぐメッシュ型 |
| 初期設定 | やや難しい | 比較的簡単 |
| ポート開放 | 必要になることがある | 基本的に不要になりやすい |
| 鍵管理 | 自分で管理する場面が多い | Tailscale側が扱いやすくしてくれる |
| 個人利用 | 設定が難しい場合がある | 始めやすい |
| 技術基盤 | OpenVPN、WireGuardなど | WireGuardベース |

---

# 5. SSHとの関係

TailscaleはSSHの代わりではありません。

役割は次のように違います。

| 技術 | 役割 |
|---|---|
| Tailscale | 端末同士が安全に通信できる経路を作る |
| SSH | 接続先PCやサーバーにログインしてコマンド操作する |

つまり、組み合わせると次のようになります。

```text
外出先ノートPC
  ↓ Tailscaleで自宅PCに到達
  ↓ SSHでログイン
自宅PC
```

コマンド例:

```bash
ssh user@自宅PCのTailscale IP
```

この構成では、自宅PCのSSHポートを直接インターネットに公開しなくて済みます。

---

# 6. リモートデスクトップとの関係

Tailscaleは、リモートデスクトップとも相性が良いです。

```text
外出先ノートPC
  ↓ Tailscale
自宅PC
  ↓ リモートデスクトップ
```

通常、リモートデスクトップを外部から使うにはポート開放が必要になることがあります。

しかし、リモートデスクトップ用のポートを直接インターネットに公開するのは危険です。

Tailscaleを使えば、Tailscaleの仮想ネットワーク内だけでリモートデスクトップを使う構成にできます。

```text
直接公開:
  インターネット
    ↓
  自宅PCのリモートデスクトップ
  → 危険になりやすい

Tailscale経由:
  外出先PC
    ↓ Tailscale
  自宅PCのリモートデスクトップ
  → 安全に設計しやすい
```

---

# 7. VPSとの関係

VPSにTailscaleを入れると、VPSも自分のプライベートネットワークに参加できます。

```text
Tailscaleネットワーク
├── 自宅PC
├── 外出用ノートPC
└── VPS
```

この構成では、次のようなことができます。

```text
外出先ノートPCからVPSへSSH
自宅PCからVPSへSSH
VPSから自宅PCへアクセス
VPSの管理用SSHをTailscale経由に限定する
```

特に、VPSのSSHをインターネットに直接公開したくない場合に便利です。

```text
通常:
  自分のPC
    ↓ SSH
  VPSの公開IP

Tailscale利用:
  自分のPC
    ↓ Tailscale
  VPSのTailscale IP
```

---

# 8. Tailscaleの重要用語

## 8.1 tailnet

tailnetとは、Tailscaleで作られる自分専用の仮想ネットワークです。

```text
tailnet = 自分のTailscaleネットワーク
```

同じtailnetに入っている端末同士が通信できます。

---

## 8.2 node

nodeとは、Tailscaleに参加している端末のことです。

```text
node = Tailscaleに参加しているPC / スマホ / サーバー
```

例:

```text
自宅PC
外出用ノートPC
スマホ
VPS
Raspberry Pi
```

---

## 8.3 Tailscale IP

Tailscale IPとは、Tailscale上で各端末に割り当てられる専用IPアドレスです。

通常の自宅LANのIPとは別に、Tailscale用のIPが付きます。

例:

```text
自宅LANのIP:
  192.168.1.10

Tailscale IP:
  100.x.y.z
```

外出先からは、Tailscale IPを使って自宅PCに接続できます。

```bash
ssh user@100.x.y.z
```

---

## 8.4 Exit Node

Exit Nodeは、Tailscale上の特定の端末を経由してインターネットへ出る機能です。

イメージ:

```text
外出先PC
  ↓ Tailscale
自宅PC / VPS（Exit Node）
  ↓
インターネット
```

たとえば、外出先PCの通信を自宅PCやVPS経由にしたい場合に使います。

ただし、最初から使う必要はありません。

まずは単純に端末同士をつなぐところから始めるのがおすすめです。

---

## 8.5 Subnet Router

Subnet Routerは、Tailscaleを入れられない機器にもアクセスしたいときに使う機能です。

たとえば、自宅にNASがあるが、NASにはTailscaleを入れられないとします。

その場合、自宅内の1台のPCやサーバーをSubnet Routerにすると、Tailscale経由でNASにアクセスできるようになります。

```text
外出先PC
  ↓ Tailscale
自宅PC（Subnet Router）
  ↓
自宅LAN内のNAS
```

ただし、Subnet Routerはネットワーク全体へのアクセスに関係するため、最初は無理に使わなくて大丈夫です。

---

# 9. Tailscaleは無料か

個人利用であれば、Tailscaleは無料で使える範囲があります。

学習用途や個人利用で、以下のような構成を試すなら、まず無料プランで十分です。

```text
外出先ノートPC ↔ 自宅PC
スマホ ↔ 自宅PC
ノートPC ↔ VPS
自宅PC ↔ VPS
Raspberry Pi ↔ ノートPC
```

## 9.1 個人利用と商用利用の違い

個人の学習・自宅利用・ホームラボ用途であれば無料で始めやすいです。

一方で、会社利用、チーム利用、本番インフラ管理などでは、有料プランを検討する必要があります。

```text
個人利用・学習用途:
  無料プランで始めやすい

会社・チーム・本番インフラ用途:
  有料プランを検討
```

---

# 10. Tailscaleのメリット

## 10.1 ポート開放を避けやすい

Tailscaleの大きなメリットは、自宅ルーターのポート開放を避けやすいことです。

```text
ポート開放あり:
  インターネットに入口を公開する

Tailscale:
  自分の端末同士だけでつながる
```

これにより、初心者でも安全に設計しやすくなります。

---

## 10.2 設定が比較的簡単

通常のVPNでは、VPNサーバー構築、証明書、ルーティングなどを考える必要があります。

Tailscaleは、各端末にインストールしてログインすることで始められるため、最初のハードルが低いです。

---

## 10.3 SSHやリモートデスクトップと組み合わせやすい

Tailscale単体で何かを操作するというより、SSHやリモートデスクトップと組み合わせると便利です。

```text
Tailscale + SSH:
  コマンド操作

Tailscale + リモートデスクトップ:
  画面操作

Tailscale + VPS:
  安全なサーバー管理
```

---

# 11. 注意点

## 11.1 アカウント管理が重要

Tailscaleはアカウントで端末を管理します。

そのため、ログインに使うGoogleアカウント、GitHubアカウント、Apple IDなどの管理が重要です。

最低限、2段階認証を有効にしておくのがおすすめです。

---

## 11.2 端末を増やしすぎない

Tailscaleに参加する端末が増えると、管理対象も増えます。

使わなくなった端末は、管理画面から削除する習慣をつけた方がよいです。

---

## 11.3 Exit NodeやSubnet Routerは慎重に使う

Exit NodeやSubnet Routerは便利ですが、通信経路やネットワーク全体へのアクセスに関わります。

最初は以下のような単純な構成から始めるのがおすすめです。

```text
外出先ノートPC
  ↓ Tailscale
自宅PC
```

慣れてきたら、必要に応じてExit NodeやSubnet Routerを学ぶとよいです。

---

# 12. 初心者におすすめの試し方

## 12.1 最初の構成

最初は、次の2台で試すのがわかりやすいです。

```text
外出用ノートPC
自宅PC
```

それぞれにTailscaleを入れて、同じアカウントでログインします。

```text
外出用ノートPC
  ↓ Tailscale
自宅PC
```

その後、自宅PCにSSHサーバーやリモートデスクトップを設定して、Tailscale IPで接続してみます。

---

## 12.2 SSHで試す

```bash
ssh user@自宅PCのTailscale IP
```

SSHが成功すれば、外出先から自宅PCをコマンド操作できます。

---

## 12.3 リモートデスクトップで試す

Windowsのリモートデスクトップや、その他のリモートデスクトップソフトを使う場合も、接続先に自宅PCのTailscale IPを指定します。

```text
接続元:
  外出用ノートPC

接続先:
  自宅PCのTailscale IP
```

---

# 13. 今回の議論のまとめ

Tailscaleは、VPNのように安全な通信経路を作りつつ、従来のVPNよりも簡単に端末同士をつなげるサービスです。

特に、SSHやリモートデスクトップと組み合わせると便利です。

```text
Tailscaleだけ:
  端末同士が安全に到達できるようになる

Tailscale + SSH:
  外出先から自宅PCやVPSをコマンド操作できる

Tailscale + リモートデスクトップ:
  外出先から自宅PCの画面を操作できる

Tailscale + VPS:
  VPSを自分のプライベートネットワークに参加させられる
```

最初に覚えるべきことは、次の3つです。

```text
Tailscale = 簡単に使えるVPNのようなサービス
TailscaleはSSHの代わりではなく、SSHを安全に通す経路になる
個人利用・学習用途なら無料プランから始めやすい
```

めがまるさんの用途で考えるなら、最初のおすすめ構成は次です。

```text
外出先ノートPC
  ↓ Tailscale
自宅PC
  ↓ SSH or リモートデスクトップ
```

この構成なら、自宅PCを直接インターネットに公開せず、比較的安全に外出先から接続しやすくなります。
