# VPNとVPSの違いまとめ

## 目的

このドキュメントは、ジュニアエンジニア向けに **VPN** と **VPS** の違いを整理するための学習メモです。

特に、以下の混同をなくすことを目的にしています。

```text
VPNとVPSは何が違うのか
SSHとはどう関係するのか
VPSを使うときにVPNは必要なのか
自宅PCへ接続するときにVPNは必要なのか
VPNサーバーとは何なのか
```

結論から言うと、VPNとVPSは名前が似ていますが、役割はまったく違います。

```text
VPN = 安全な通信経路を作る仕組み
VPS = インターネット上で借りる仮想サーバー
```

---

# 1. まず全体像

## 1.1 VPN・VPS・SSHの関係

今回の議論では、VPNとVPSだけでなく、SSHも関係してきました。

3つの役割を一言で整理すると、次のようになります。

| 用語 | 一言でいうと | 役割 |
|---|---|---|
| SSH | サーバーやPCにログインする方法 | 接続・操作の手段 |
| VPN | 安全な通信経路 | ネットワークに入るための通路 |
| VPS | 借りる仮想サーバー | 接続先・サーバーそのもの |

図で表すと、次のようなイメージです。

```text
SSH = 入る方法
VPN = 安全な道
VPS = 入る先・使う場所
```

もう少し具体的にすると、次のようになります。

```text
自分のPC
  ↓ SSH
VPS

自分のPC
  ↓ VPN
自宅ネットワーク
  ↓ SSH
自宅PC
```

# 2. VPNとは

## 2.1 VPNの意味

VPNは **Virtual Private Network** の略です。

日本語っぽく言うと、

```text
インターネット上に、安全な専用トンネルを作る仕組み
```

です。

VPNを使うと、外出先のPCやスマホから、自宅や会社などの別ネットワークに安全に入ることができます。

## 2.2 VPNのイメージ

たとえば、外出先のノートPCから自宅ネットワークに入りたい場合、VPNは次のように働きます。

```text
外出先のノートPC
  ↓ VPN接続
自宅ネットワーク
  ↓
家のPC / NAS / Raspberry Pi など
```

VPNを使うと、外出先にいても、まるで自宅Wi-Fi内にいるような状態を作れます。

## 2.3 VPNは「接続先」ではなく「通路」

VPNはサーバーそのものではありません。

VPNは、特定のネットワークへ安全に入るための **通路** です。

```text
VPN = 安全な通路
```

たとえば、SSHやリモートデスクトップを安全に使うために、VPNを通して接続することがあります。

```text
外出先PC
  ↓ VPN
自宅ネットワーク
  ↓ SSH または リモートデスクトップ
自宅PC
```

# 3. VPSとは

## 3.1 VPSの意味

VPSは **Virtual Private Server** の略です。

日本語っぽく言うと、

```text
インターネット上で借りる、自分専用のように使える仮想サーバー
```

です。

VPSは、通信経路ではなく、**サーバーそのもの**です。

## 3.2 VPSのイメージ

VPSは、事業者が持っている大きな物理サーバーを仮想化して、複数のユーザーに貸し出しているものです。

```text
大きな物理サーバー
├── VPS A：ユーザーAが使う
├── VPS B：ユーザーBが使う
├── VPS C：ユーザーCが使う
└── VPS D：ユーザーDが使う
```

ユーザーから見ると、自分専用のLinuxサーバーを1台借りているように扱えます。

## 3.3 VPSは「接続するための仕組み」ではない

VPSは、接続するための仕組みではありません。

正しくは、

```text
VPS = 接続先になるサーバー
SSH = VPSへ接続するための方法
```

です。

例:

```text
自分のPC
  ↓ SSH
VPS
```

この場合、SSHが接続方法で、VPSが接続先です。

# 4. VPNとVPSの違い

## 4.1 比較表

| 項目 | VPN | VPS |
|---|---|---|
| 正式名称 | Virtual Private Network | Virtual Private Server |
| 日本語イメージ | 仮想的な専用ネットワーク | 仮想的な専用サーバー |
| 役割 | 安全な通信経路を作る | サーバーを借りる |
| 実体 | ネットワークの仕組み | 仮想マシン / 仮想サーバー |
| 主な用途 | 自宅・会社ネットワークへの安全な接続 | Webアプリ公開、DB運用、サーバー構築 |
| SSHとの関係 | SSHを安全に通す経路になる | SSHでログインする対象になる |
| たとえ | 安全な専用道路・トンネル | 借りた作業部屋・サーバー部屋 |

## 4.2 一言で整理

```text
VPN = 道
VPS = 場所
SSH = 入る方法
```

たとえるなら、以下のような関係です。

```text
VPS:
  インターネット上に借りた作業部屋

SSH:
  その作業部屋に入るための鍵付きドア

VPN:
  離れた場所から安全に通れる専用トンネル
```


---

# 5. SSHとの関係

## 5.1 VPSを使うときのSSH

VPSを借りた場合、多くのケースではSSHでログインして操作します。

```text
自分のPC
  ↓ SSH
VPS
  ↓
アプリやDBを動かす
```

例:

```bash
ssh user@VPSのIPアドレス
```

このとき、VPNは必須ではありません。

多くのVPSは、インターネット経由でSSH接続できるように作られています。

## 5.2 VPNを使うときのSSH

VPNは、SSHを安全に通すための経路として使われることがあります。

たとえば、外出先から自宅PCにSSHしたい場合です。

```text
外出先のノートPC
  ↓ VPN
自宅ネットワーク
  ↓ SSH
家のPC
```

この場合、

```text
VPN = 自宅ネットワークに入るための通路
SSH = 家のPCへログインする方法
家のPC = 接続先
```

です。


---

# 6. VPSを使うときにVPNは必要か

## 6.1 通常は不要

通常、VPSに接続するためにVPNは必要ありません。

多くの場合、次のようにSSHで直接接続します。

```text
自分のPC
  ↓ SSH
インターネット上のVPS
```

例:

```bash
ssh user@xxx.xxx.xxx.xxx
```

この場合に必要なのは、VPNではなくSSHです。

## 6.2 VPNが必要になる場合もある

ただし、構成によってはVPSとVPNが一緒に使われることもあります。

代表的なケースは以下です。

| ケース | 説明 |
|---|---|
| VPSをVPNサーバーとして使う | VPS上にWireGuardやOpenVPNを構築する |
| VPSへのSSHをVPN内だけに制限する | セキュリティのために、VPN接続者だけがSSHできるようにする |
| VPSから自宅や社内ネットワークへ接続したい | VPSと別ネットワークをVPNでつなぐ |

# 7. 「別のサーバー」はすべてVPSなのか

## 7.1 すべてがVPSではない

自分のPCとは別の場所にあるサーバーが、すべてVPSというわけではありません。

```text
別のサーバーとして機能しているもの
  ≠ すべてVPS
```

VPSは、サーバーの種類の1つです。

## 7.2 サーバーの種類

| 種類 | 説明 | VPSか |
|---|---|---|
| 自宅PCサーバー | 自宅PCでWebサーバーなどを動かす | いいえ |
| 物理専用サーバー | 物理マシン1台を丸ごと借りる | いいえ |
| VPS | 物理サーバーを仮想化して分割したサーバー | はい |
| クラウドVM | AWS EC2などの仮想マシン | VPSに近い |
| レンタルサーバー | Webサイト置き場を借りる共有型サービス | 基本は違う |
| PaaS | Vercel、Render、Railwayなど | 違う |
| コンテナ | Dockerコンテナなど | 違う |

## 7.3 VPSかどうかの見分け方

以下に当てはまる場合、VPSまたはクラウドVMに近いです。

```text
自分専用の仮想マシンがある
OSにSSHログインできる
Linuxの設定を自分で変更できる
Nginx、Docker、DBなどを自分で入れられる
CPU、メモリ、ストレージのプランがある
```

逆に、以下のようなサービスはVPSではなく、PaaSやホスティングサービスに近いです。

```text
GitHubと連携してデプロイするだけ
OS管理はしない
サーバー設定を直接触らない
SSHログインしない
```

例:

```text
Vercel
Netlify
Firebase Hosting
Render
Railway
```

# 8. VPNサーバーとは

## 8.1 VPNサーバーの意味

VPNサーバーとは、

```text
VPN接続を受け付けて、別のネットワークへ安全に入れるようにする入口役のサーバー
```

です。

VPNを「安全なトンネル」と考えると、VPNサーバーはトンネルの入口・出口を管理する存在です。

## 8.2 VPNサーバーとVPNクライアント

VPNにも、SSHと同じようにサーバー側とクライアント側があります。

| 役割 | 意味 | 例 |
|---|---|---|
| VPNサーバー | VPN接続を受け付ける側 | 自宅ルーター、VPS、会社のVPN装置 |
| VPNクライアント | VPNに接続する側 | 外出先PC、スマホ、タブレット |

図にすると次の通りです。

```text
VPNクライアント
  ↓ 接続する
VPNサーバー
  ↓ ネットワークに入れる
目的のPCやサーバー
```

## 8.3 SSHサーバーとの違い

| 項目 | SSHサーバー | VPNサーバー |
|---|---|---|
| 目的 | そのPC/サーバーにログインさせる | ネットワークに参加させる |
| 操作対象 | 基本は1台のPC/サーバー | ネットワーク全体 |
| 接続後 | コマンド操作できる | LAN内にいるように通信できる |
| 用途 | サーバー管理、コマンド操作 | 自宅/会社ネットワークへの安全な接続 |

一言でいうと、次のようになります。

```text
SSHサーバー = そのPCにログインさせる
VPNサーバー = そのネットワークに入らせる
```

# 9. 自宅PCへ接続するならSSHだけでよいのか

## 9.1 SSHだけでも可能

外出先から自宅PC 1台にコマンドで入りたいだけなら、SSHだけでも可能です。

```text
外出先のノートPC
  ↓ SSH
インターネット
  ↓
自宅ルーター
  ↓ ポート開放
家のPC
```

ただし、この場合は自宅ルーターでポート開放が必要になることが多いです。

例:

```text
外部ポート 2222
  ↓
家のPCの22番ポート
```

外出先からは、次のように接続します。

```bash
ssh -p 2222 user@自宅のグローバルIP
```

## 9.2 SSHだけで外部公開する場合の注意

SSHだけで外部から接続できるようにすると、自宅PCのSSH入口がインターネット側から見える状態になります。

```text
インターネット上の誰か
  ↓
自宅ルーターのSSH入口までは到達できる
```

そのため、以下のような対策が重要です。

```text
公開鍵認証を使う
パスワード認証を無効化する
不要なポートを開けない
ファイアウォールを設定する
ログを確認する
秘密鍵を安全に保管する
```

SSHだけでも可能ですが、初心者が外部公開する場合は慎重さが必要です。

## 9.3 VPN + SSHの場合

VPNを使うと、先に自宅ネットワークへ入り、その後でSSHできます。

```text
外出先のノートPC
  ↓ VPN
自宅ネットワークに入る
  ↓ SSH
家のPC
```

この場合、外出先PCは、自宅Wi-Fi内にいるような状態になります。

そのため、家のPCのローカルIPに対してSSHできます。

```bash
ssh user@192.168.1.10
```

## 9.4 SSHだけとVPN + SSHの違い

| 比較 | SSHだけ | VPN + SSH |
|---|---|---|
| できること | 家のPC 1台に入る | 自宅ネットワーク全体に入れる |
| 外部から見える入口 | SSH | VPN |
| 自宅PCのSSH公開 | 外部公開される | 外部公開しなくてよい |
| 接続対象 | 基本は1台 | PC、NAS、Raspberry Piなど複数 |
| 設定 | シンプルだが慎重さが必要 | 最初は少し複雑 |
| 安全性 | 設定次第 | 安全に設計しやすい |

# 10. 具体的な利用シーン

## 10.1 WebアプリをVPSで公開する

VPSを借りてWebアプリを公開する場合は、主にSSHを使います。

```text
自分のPC
  ↓ SSH
VPS
  ↓
Docker / Nginx / DB / アプリを設定
```

この場合、VPNは通常不要です。

## 10.2 外出先から自宅PCを安全に操作する

自宅PCを外出先から操作したい場合は、VPNが関係します。

```text
外出先PC
  ↓ VPN
自宅ネットワーク
  ↓ SSH または リモートデスクトップ
家のPC
```

自宅PCを直接インターネットに公開せずに済むため、安全寄りです。

## 10.3 VPSをVPNサーバーとして使う

VPS上にVPNサーバーを構築することもできます。

```text
自分のPC / スマホ
  ↓ VPN
VPS上のVPNサーバー
  ↓
インターネット
```

この構成では、自分の通信をVPS経由にできます。

また、複数端末を同じVPNに参加させることで、離れた場所の端末同士を安全につなぐこともできます。

## 10.4 会社ネットワークに入ってからサーバーへSSHする

会社では、サーバーをインターネットに直接公開せず、VPN内からしか接続できないようにしていることがあります。

```text
自宅PC
  ↓ VPN
会社ネットワーク
  ↓ SSH
社内サーバー
```

この場合、先にVPN接続しないとSSHできません。

# 11. Tailscaleについて

## 11.1 Tailscaleとは

Tailscaleは、VPNに近い仕組みを簡単に使えるサービスです。

難しいVPNサーバー構築やルーターのポート開放をしなくても、自分の端末同士を安全につなぎやすくなります。

```text
外出先PC
  ↓ Tailscale
家のPC
  ↓ SSH または リモートデスクトップ
```

## 11.2 個人用途では現実的な選択肢

外出先から自宅PCに安全に入りたい場合、初心者がいきなりルーターのポート開放を行うより、Tailscaleのような仕組みを使う方が安全で扱いやすいことがあります。

```text
SSHだけ:
  自宅PCのSSH入口を外に出す必要がある

Tailscale + SSH:
  自宅PCのSSH入口を直接インターネットに公開しなくてよい
```

# 12. よくある誤解

## 12.1 VPSは別のサーバーに接続するための仕組み？

違います。

```text
誤:
  VPS = 別のサーバーに接続するための仕組み

正:
  VPS = インターネット上で借りるサーバーそのもの
```

接続するための仕組みはSSHなどです。

## 12.2 VPNは別のネットワークにアクセスする仕組み？

これはかなり正しいです。

より正確には、

```text
VPN = 別のネットワークに安全にアクセスするための通信経路を作る仕組み
```

です。

## 12.3 VPSを使うならVPNが必要？

通常は不要です。

```text
自分のPC
  ↓ SSH
VPS
```

で接続できます。

VPNが必要になるのは、VPSをVPNサーバーとして使う場合や、VPN内からしかVPSへ接続できないようにしている場合などです。

## 12.4 自宅PCにSSHするならVPNは不要？

目的によります。

```text
家のPC 1台にSSHするだけ
  → SSHだけでも可能

自宅ネットワーク全体に安全に入りたい
  → VPNが向いている

SSHを直接インターネットに公開したくない
  → VPN + SSH が安全寄り
```

# 13. 最終まとめ

## 13.1 3つの役割

最後に、SSH・VPN・VPSをまとめると次のようになります。

| 用語 | 役割 | 例 |
|---|---|---|
| SSH | サーバーやPCにログインする方法 | VPSへログイン、自宅PCへログイン |
| VPN | 安全なネットワーク経路 | 自宅ネットワーク、会社ネットワークへ入る |
| VPS | 借りる仮想サーバー | Webアプリ公開、DB運用、VPNサーバー構築 |

## 13.2 覚え方

```text
SSH = 入る方法
VPN = 安全な道
VPS = 入る先・使う場所
```

または、家にたとえるなら、

```text
SSH = 部屋に入るための鍵付きドア
VPN = 建物まで安全に行く専用トンネル
VPS = 借りた作業部屋
```

## 13.3 実務的な使い分け

| やりたいこと | 主に使うもの |
|---|---|
| VPSにログインして設定したい | SSH |
| VPSでWebアプリを公開したい | VPS + SSH |
| 外出先から自宅PCに安全に接続したい | VPN + SSH / リモートデスクトップ |
| 自宅PCを直接インターネットに公開したくない | VPN / Tailscale |
| 会社ネットワーク内のサーバーに入りたい | VPN + SSH |
| 自分でVPNサーバーを立てたい | VPS + VPNソフトウェア + SSH |

# 14. 学習のおすすめ順

ジュニアエンジニアとして理解するなら、次の順番で学ぶと整理しやすいです。

```text
1. SSHの基本を理解する
2. VPSを借りると何ができるか理解する
3. VPSへSSH接続する
4. VPNはネットワークに入るための仕組みだと理解する
5. 自宅PCへの接続で SSHだけ / VPN + SSH の違いを理解する
6. Tailscaleなどで安全な接続方法を試す
7. 必要になったらWireGuardやOpenVPNなどを学ぶ
```

最初からすべてを細かく理解する必要はありません。

まずは、次の3つを押さえると十分です。

```text
SSH = サーバーに入る方法
VPN = ネットワークに安全に入る道
VPS = インターネット上で借りるサーバー
```
